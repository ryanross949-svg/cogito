About TURING

Methodology and design principles.

TURING is a cognitive training simulator designed to install the algorithmic thinking patterns used by computer scientists and logicians. It trains you to break down complex processes into discrete, unambiguous steps.

// Core Pedagogy

The system uses four evidence-based learning mechanisms, applied in concert:

  • Spaced Repetition (SM-2): Cards reappear at expanding intervals based on recall quality.
  • Active Recall: Every drill requires generation, not recognition.
  • Interleaving: Drills mix concepts across modules.
  • Elaboration: Arena problems and Project OMEGA require applying concepts to novel scenarios.

// The Ten Axes

  • AXIOM — Boolean Logic (AND/OR/NOT, Truth Tables, De Morgan's Laws)
  • STATE — State Machines (FSMs, Turing Machines, Transitions)
  • HALT — The Halting Problem (Decidability, Turing Completeness)
  • COMPLEX — Big O Notation (Time/Space Complexity, O(1), O(n), O(log n))
  • CRYPTO — Cryptography (One-way functions, RSA, Hashing)
  • SORT — Sorting & Searching (Binary search, Quicksort, P vs NP)
  • GRAPH — Graph Theory (Nodes, Edges, Dijkstra, Trees)
  • PARADOX — Logic Paradoxes (Liar's Paradox, Gödel's Incompleteness)
  • BITS — Information Theory (Entropy, Compression, Shannon)
  • MIND — Philosophy of Mind (Searle's Chinese Room, Turing Test)
